# دليلك ايجنت — توثيق المشروع

آخر تحديث: 2026-07-17

## نظرة عامة

"دليلك" (Dalilak) تطبيق Next.js 14 (App Router) لدليل الخدمات الحكومية اللبنانية. هذا الملف يوثّق حالة المشروع بعد جولة تدقيق شاملة على كل الخدمات والنماذج القابلة للتحميل.

- المستودع: `dalilak_AI_transfer/frontend`
- نظام الألوان: أحمر `#8B1A1A`، عنابي `#6b2737`، خط Cairo، RTL عربي بالكامل (ممنوع استخدام الأزرق/الأخضر كألوان أساسية)
- الحالة الحالية: **367 خدمة موثّقة عبر 58 فئة**

## آخر جولة عمل: تدقيق شامل للروابط والنماذج

بطلب من المستخدم: التأكد أن كل خدمة لها نموذج قابل للتحميل فعلياً وصالح.

### النتائج

**allServices.ts** — تم فحص 70 نطاقاً (domain) فريداً عبر DNS + HTTP مع User-Agent متصفح حقيقي. تم إصلاح:
- 15 نطاق ميت (NXDOMAIN) واستُبدل بنطاقات حية موثّقة: `interior→moim`, `mof→finance`, `public-health→moph`, `moe(طاقة)→energyandwater`, `conseilideal→statecouncil(http)`, `isa→insurancecommission`, `moc→tra`, `moyslebanon→minijes(http)`, `pressorder→pressorderlebanon`, `refugees→ministryofdisplaced(http)`, `regie.gov.lb→regie.com.lb`, `southcouncil→councilforsouth`, `cadastre→finance`, `mfa.gov.lb→mfa.gov.lb/arabic/home`
- حقلَي رابط مُدمَجين بشكل خاطئ (نصّان ملتصقان في حقل واحد)
- رابط وزارة مصنَّف خطأً

النتيجة النهائية: **0 نطاقات ميتة** من أصل 70، جميعها verified عبر esbuild + UTF-8.

**procedures.ts (FORMS_DATA)** — 15 نموذجاً منسّقاً يدوياً. تم إصلاح 3 نطاقات ميتة (`interior→moim`, `deeds→finance`, `publicworks→transportation`) ومرفق PDF ميت واحد (تحويله لرابط صفحة نماذج عامة بدل ملف 404).

النتيجة: **15/15 نموذجاً يعمل فعلياً (HTTP 200 مؤكد)**.

**allTransactions.ts (795 رابط Wayback)** — هذا الجزء يحتاج اهتماماً خاصاً، انظر القسم التالي.

### مشكلة جوهرية مكتشفة: روابط Wayback المُختلقة

مجموعة الـ794 رابط PDF المؤرشفة عبر Wayback Machine (أُضيفت في جلسة سابقة قبل هذه المحادثة تحت اسم "850 dawlati links") تبيّن أن **نسبة كبيرة منها (~50%+) قد تكون غير حقيقية** — أي روابط تم بناؤها بتخمين اسم الملف دون التحقق الفعلي من أرشفتها.

**دليل ملموس**: مجلد `dawlati.gov.lb/documents/10180/475468` يحتوي 84 رابطاً في قاعدة البيانات لدينا، لكن فهرس Wayback الحقيقي (عبر CDX Search API) يؤكد وجود 4 فقط فعلياً (نسبة تزييف ~95% في هذا المجلد تحديداً). عيّنة عشوائية أوسع (~20 رابط) من كامل المجموعة أظهرت نسبة تزييف ~50-55%.

تم بالفعل: تحليل مجلد `10180/475468` بالكامل مقابل فهرس Wayback الحقيقي، وإضافة النموذج الحقيقي الوحيد الناقص (`MBK9-08`) الذي كان مؤرشفاً فعلياً لكنه غير مربوط في قاعدة البيانات.

**سبب عدم إصلاح البقية الآن**: التحقق يتطلب طلب واحد فقط في كل مرة (Internet Archive تحظر الطلبات المتزامنة من هذه البيئة)، وفحص 794 رابطاً بشكل تسلسلي يستغرق ساعات طويلة. بدل إصلاح جزئي أو حذف عشوائي قد يُتلف روابط حقيقية، تم بناء حل دائم:

### الحل: مهمة مجدولة يومية (`wayback-forms-audit`)

- **الآلية**: `scripts/wayback_audit_daily.py` يفحص 50 رابطاً يومياً بشكل تسلسلي بطيء (احتراماً لحد Internet Archive)، يحفظ النتيجة (REAL/FAKE) في `scripts/wayback_audit_state.json` بعد كل رابط فرداً كي لا يضيع أي تقدّم عند انقطاع.
- **الجدولة**: تعمل تلقائياً يومياً (cron `0 9 * * *`) دون الحاجة لأي متابعة يدوية.
- **متى تكتمل**: بعد ~16 يوماً (794 ÷ 50) سيكون كل رابط مصنّفاً REAL أو FAKE.
- **بعد الاكتمال**: يجب مراجعة `wayback_audit_state.json` يدوياً (أو عبر جلسة Claude لاحقة) قبل حذف أي إدخالات من `allTransactions.ts`، لأن حذف عناصر يغيّر فهارس المصفوفة `_R` التي يعتمد عليها القاموس `_P`.
